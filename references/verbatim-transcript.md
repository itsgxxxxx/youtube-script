# Verbatim Transcript

## Goal

默认目标是输出尽量贴近原视频说法的高保真 transcript，而不是整理摘要。

## Source Of Truth

- YouTube transcript / captions 是权威底稿。
- 如果只有自动字幕，也仍以自动字幕为底稿，只做最小必要清洗。
- 不要因为字幕显得杂乱就主动重写成总结稿。

## Allowed Changes

`verbatim_transcript` 模式下，只允许以下修改：

- 删除明显重复的渐进式自动字幕内容
- 删除少量口头禅，例如 `uh`、`um`、`you know`
- 补基础标点和断句
- 修正明显的 ASR 连写、断词、重复词错误
- 在不改变内容顺序的前提下分段
- 将 `>>`、连字符轮次提示、重复引导词等视为分段信号并清理出最终正文
- 保留对后文理解有帮助的背景铺垫句、引语引入句、例子前置说明和承上启下的上下文句

## Forbidden Changes

`verbatim_transcript` 模式下，禁止：

- 总结或概括 2 分钟以上的连续内容
- 改写观点或压缩论证
- 为了“更好看”而减少信息密度
- 在没有依据时补说话人真实身份
- 把多轮来回对话揉成单段说明
- 把自动字幕里的 `>>` 等轮次提示符原样保留在最终稿中（除非用户明确要求保留原始转写痕迹）
- 因为某句话看起来像铺垫、引子、例子前说明或转场句，就把它省略掉

## Density Standard

- 时间戳要足够密集，便于用户回看原视频。
- 对谈类内容，通常每 20-60 秒应有一个正文块。
- 如果某段连续讲话超过 90 秒，也不应只保留一句总结。
- 宁可略显啰嗦，也不要稀释原始内容。
- 即使不标注说话人姓名，也要让发言主体切换在排版上清晰可见；明显问答往返通常应拆成独立段落。
- 如果某条关键信息依赖前一句背景铺垫才能成立，应优先一起保留，而不是只留下结论句。
