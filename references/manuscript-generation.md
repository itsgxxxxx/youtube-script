# Manuscript Generation

## Goal

目标是让宿主模型直接从 cleaned transcript 生成一篇接近人工整理稿的完整文稿，而且应覆盖整个可用 transcript，而不是输出字幕清洗稿或节选摘要。

## Prompt Contract

在生成正文时，宿主模型应遵守以下要求：

1. 把 cleaned transcript 整理成一篇可阅读的完整文稿。
2. 使用 `H1/H2/H3` 标题组织结构。
3. 删除明显语气词、重复、卡顿和字幕碎裂感。
4. 合并短句，改进断句、标点和语法。
5. 保留原始讨论顺序和主要措辞，不新增事实、评论或背景解释。
6. 默认不要明显压缩内容；如果原文一节很长，输出也应保持相近的信息密度。
7. 默认处理整个 cleaned transcript；除非底稿本身缺失，否则不要只写前半段、不要只挑重点段落。
8. 多人对话通过段落组织体现问答关系；除非身份非常可靠，否则不要强行标人名。
9. `timestamp_mode=sparse` 时，只在大章节前保留少量时间点；`timestamp_mode=none` 时不保留时间戳。
10. 不要输出元数据、目录、要点、总结、附录、解释性说明。
11. 正文必须直接开始，不能插入“以下是整理稿”“我为你生成了文稿”等额外话术。

## Style Target

- 读感像整理过的访谈稿，而不是字幕串接。
- 标题要像编辑起的主题标题，而不是导航标签。
- 段落应顺畅、自然、清晰，避免生硬逐句直译。
- 允许轻度书面化，但不允许改写论点或偷换原意。
- 更接近“整理过的完整文稿”，而不是“总结过的长摘要”。
- 完成度应接近“把整场谈话整理成可读长文”，而不是“从长谈里摘出一部分重点”。

## Recommended System Framing

可把生成任务理解为：

- 先做内容编辑，再做格式编辑
- 先确保忠实，再追求可读
- 先去掉字幕噪音，再组织文稿结构
- 先保证覆盖全程，再决定章节切分

## Output Skeleton

```markdown
# 标题

## [可选稀疏时间点] 主题标题

自然段

自然段
```
